# 校园网断网自动重连工具

适用场景：上网需要在浏览器认证页面输账号密码，隔一段时间被踢下线、要重新手动登录。
本工具在后台自动检测，一断网就替你重新登录，**全程无窗口、不打扰你前台工作**，无需任何操作。

> 已内置**深澜（Srun）认证系统**的加密登录算法，默认对应 **杭电校园网（login.hdu.edu.cn）**。
> 同校同学开箱即用，只需填账号密码。其它学校见文末「其它学校 / 其它认证系统」。

---

## 一、它是怎么保护你的（两层保险 + 全程无窗口）

装好后会有两个后台任务一起工作，互相独立，一个坏了另一个顶上：

| | 主程序 | 看门狗 |
|---|---|---|
| 作用 | 主力：每 **30 秒**检测一次，断网立即重连 | 兜底：每 **3 分钟**独立巡查一次，万一主程序卡死/退出，由它救回网络 |
| 是否常驻 | 是，一直在后台 | 否，每 3 分钟被叫醒、跑 1~2 秒就退出 |

**三重保障，掉线也能自己回来：**

1. **主程序自愈**：万一被意外掐断（比如睡眠/休眠唤醒后进程没了），系统每 5 分钟会自动把它重新拉起；
2. **看门狗兜底**：用的是「定时」触发，**睡眠/休眠唤醒后也能照常巡查**，不会失效；
3. **崩溃重启**：主程序若崩溃，Windows 还会在 1 分钟内再拉它一把。

> 🪟 **全程无窗口**：两个任务都经由 `静默启动.vbs` 在后台隐藏启动，**不会再出现一闪而过的终端黑框**。

## 二、文件说明

| 文件 | 作用 |
|------|------|
| `config.json` | **配置文件：填你的账号密码（唯一需要你动的文件）** |
| `测试登录.bat` | 双击：强制登录一次，验证账号密码对不对 |
| `安装开机自启.bat` | 双击：注册开机自启并立即开始后台运行 |
| `卸载开机自启.bat` | 双击：彻底停止并删除（主程序 + 看门狗一起删） |
| `测试运行.bat` | 双击：只检测一次当前网络状态（不常用） |
| `keeper.ps1` | 主脚本（核心逻辑，一般不用动） |
| `install.ps1` / `uninstall.ps1` | 上面两个 bat 实际调用的脚本 |
| `静默启动.vbs` | **后台隐藏启动器**，让 keeper 无窗口运行。⚠️ 别删、别改名、别移走、**也别去双击它** |
| `运行日志.txt` | 自动生成：**事件日志**（断网、登录、报错才记一行） |
| `最近检测.txt` | 自动生成：**心跳**，永远只有一行，显示最后一次检测时间和结果 |

> 关于 `静默启动.vbs`：它必须和 `keeper.ps1` 待在**同一个文件夹**里。它没有界面，**双击只会在后台多起一个不受管理的重复进程**——要启动/重装，永远用 `安装开机自启.bat`。

## 三、使用步骤（杭电同学三步搞定）

### 第 1 步：填账号密码

用记事本打开 `config.json`，改最上面两行：

```json
"username": "在这里填你的学号",
"password": "在这里填你的密码",
```

其余不用动。（杭电的 `base_url` 已经预填好。）

### 第 2 步：测试登录

双击 `测试登录.bat`，看弹出的窗口：

- 出现 `error":"ok` 或「登录成功」→ **账号密码正确，成功** ✅
- 出现 `password` / `E2xxx` 之类报错 → 账号或密码错了，回第 1 步检查
- 出现获取 challenge 失败 → 当前网线本身没通，或不在校园网内

### 第 3 步：开机自启

测试通过后，双击 `安装开机自启.bat`。**会弹出一次 UAC（用户账户控制）请求管理员权限——点「是」即可**（注册系统级开机任务需要管理员权限，无需输入任何密码）。
看到「已创建两个任务并开始运行（经 静默启动.vbs 拉起，全程无终端窗口）」就大功告成。
之后**开机即在后台自动运行——不需要登录 Windows、锁屏状态也照常工作**，断网自动重连，你什么都不用管，也不会看到任何窗口弹出。

## 四、怎么确认它在正常工作

- **想随时确认它还活着**：打开 `最近检测.txt`，看时间是不是 30 秒以内的。例如：
  `最后检测: 2026-06-15 00:28:16  状态: 网络正常  (常驻进程)`
- **想看它干过什么**：打开 `运行日志.txt`。注意它**只在状态变化时记录**（断网、登录、恢复），网络一直正常时不会刷屏，这是正常的。
- **想看真实重连效果**：等下次它自然断网，回头看 `运行日志.txt` 里会出现「检测到掉线 → 登录成功，网络已恢复」。
  ⚠️ 不要为了测试自己去认证页点「注销」——尤其是远程操作这台电脑时，万一没立刻连回来会很被动。让它自然断网来验证最安全。

---

## 五、改了配置后要重启

`config.json`（包括换密码）和 `keeper.ps1` 都是**程序启动那一刻读进内存的**，改完文件不会自动生效。
让新配置生效的办法：**先双击 `卸载开机自启.bat`，再双击 `安装开机自启.bat`**（两步各会弹一次 UAC，点「是」即可；或直接重启电脑）。
（`静默启动.vbs` 不用动，它只是个启动器，不读配置。）

---

## 六、不想用了 / 卸载

双击 `卸载开机自启.bat`（同样会弹一次 UAC，点「是」），主程序和看门狗会一起被删干净，后台不留任何东西，开机也不再自启。

## 七、常见问题

- **会不会弹黑框/终端窗口打扰我？**：不会。两个任务都经 `静默启动.vbs` 隐藏运行，全程无窗口。如果你仍看到每隔几分钟一闪而过的窗口，说明是旧版本——重新双击一次 `安装开机自启.bat` 即可（新版会改用静默启动）。
- **任务起不来 / 装完没反应**：检查 `静默启动.vbs` 是否还在，且和 `keeper.ps1`、`install.ps1` 在同一文件夹。被删/被移走会导致后台任务启动失败，找回来或重新安装即可。
- **测试登录报错 `E2xxx` 或提示位置/ac_id 不对**：不同校区、不同楼栋的 `ac_id` 可能不一样。正常运行时脚本会自动识别，一般不用管；如果测试登录这步报错，断网时双击 `测试运行.bat`，`运行日志.txt` 里会打印认证页地址，把里面 `ac_id=` 后面的数字填到 `config.json` 的 `srun.ac_id`。
- **能上网，但 `最近检测.txt` 一直显示「被认证页拦截」**：可能你的网络劫持了默认探测地址。把 `config.json` 里 `check.probe_url` 换成 `http://connect.rom.miui.com/generate_204`，并把 `expected_content` 留空 `""`，然后重装。
- **账号有同时在线设备数限制**：自动登录可能把你别的设备顶下线，这是认证系统的规则，脚本无法绕过。
- **双击 bat 闪退或提示脚本被禁止运行**：bat 里已带 `-ExecutionPolicy Bypass`，正常不会拦。若仍被拦，是公司/学校策略限制，需联系管理员。
- **睡眠/休眠唤醒后还会工作吗**：会。看门狗用的是定时触发、唤醒后能补跑；主程序若被睡眠掐断，也会在 5 分钟内自愈重启。
- **这台电脑会自动重启吗（远程使用注意）**：不用担心。两个任务以 **SYSTEM 身份、开机即运行**，**不需要任何人登录 Windows**。所以机器自动重启、停在登录界面、锁屏状态下掉网，都会照常自动重连——这正是远程使用这台电脑的关键。安装/卸载需要管理员权限（双击 bat 会弹一次 UAC，点「是」即可）。

## 八、其它学校 / 其它认证系统

- **同样是深澜（Srun）但不是杭电**：认证页地址通常长得像 `xxx.edu.cn` 带 `/srun_portal` 或 `rad_user_info`。把 `config.json` 里 `srun.base_url` 改成你们的认证服务器地址即可（比如 `https://你们的认证域名`）。
- **不是深澜**（锐捷 ePortal、Dr.COM、天翼等）：本工具也支持「通用模板」模式，但需要先抓一次真实登录请求来配置 `login` 段，且若密码是网页加密提交的需另行适配。这部分较复杂，建议找懂的人帮忙改 `config.json` 的 `portal_type` 与 `login` 段。

---

## 九、安全提醒

密码以**明文**保存在 `config.json` 里。**把本工具发给别人前，请先把 `config.json` 里的账号密码改成对方自己的，或清空**，不要把含有你密码的文件夹直接分享出去。
